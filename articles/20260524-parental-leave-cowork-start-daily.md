## はじめに

今日は育休最終日です。明日から仕事復帰します。

育休中に自分なりに試行錯誤して最適化してきた日常の仕組みについて、書き残しておこうと思います。

育休に入って最初に感じたのは「時間があるようで、全然ない」という感覚でした。

我が家は**完全母乳**だったので、授乳は妻担当。その分、自分はおむつ替え・寝かしつけ・沐浴・家事・ご飯…と、授乳以外のことを全部やる、というスタンスで育休に臨みました。**「妻をできるだけサポートし、自分のできることを全うする」** をモットーに。

でも実際やってみると、授乳のタイミングに合わせて動く必要があったり、寝不足でぼーっとした頭で「今日何するんだっけ」を考えたりと、なかなかにカオスな日々でした。

そんな育休期間を、ただ育児するだけでなく「AI のキャッチアップにも使いたい」と思っていました。エンジニアとして、AI ツールの進化についていくことは仕事復帰後にも直結する。でも本格的な開発をするような余裕はない。そこで目をつけたのが、日常生活そのものを AI で最適化することでした。

そんな中で試し始めたのが **Cowork**（Claude のデスクトップアプリ）と、その中で自分でカスタマイズした **`start-daily` スキル** です。

この記事では、育休中という「予測不可能な毎日」でどう Cowork を活用し、何を最適化してきたか、特に `start-daily` スキルを中心に紹介します。

## Coworkとは

[Cowork](https://claude.ai/download) は、Anthropic が提供する Claude のデスクトップアプリです。

ただのチャットアプリではなく、**スキル**と呼ばれる独自のプロンプトセットを登録できるのが特徴。繰り返し行う作業をスキルとして定義しておくと、`/start-daily` のように呼び出すだけで複雑なタスクを自動実行してくれます。

Google カレンダーや Notion、Google Drive などの外部サービスとも MCP（Model Context Protocol）で連携でき、「AI がいろんなサービスを横断して動いてくれる」感覚が味わえます。

### なぜ Claude Code ではなく Cowork を選んだか

普段の仕事では Claude Code（CLI ツール）をメインで使っています。コードを書く・リファクタする・調査するといった開発タスクは Claude Code が圧倒的に速い。

でも育休中の日常管理に Claude Code を使うのは、**ちょっと違うな**と思いました。

ターミナルを開いてプロンプトを打つ、という体験は仕事モードのスイッチが入ってしまう。育休中は「エンジニアとして考える」場面を意図的に減らして、生活に集中したかった。

Cowork はチャット感覚で話しかけるだけで動いてくれるので、仕事とプライベートの切り替えがしやすい。**同じ Claude を、ツールを変えることで使い分ける**、というのが自分にとってちょうどよかったです。

## `start-daily` スキルについて

`start-daily` は、**毎朝のデイリーノートを自動で作成するスキル**です。

チャットで「/start-daily」や「今日のスケジュール作って」と話しかけるだけで起動します。

![start-daily実行時の画面](/images/parental-leave-cowork-start-daily/start-daily-execution.png)

### やってくれること

大きく分けると以下の6ステップを自動でこなしてくれます。

**① タスクを3つのソースから収集する**

- `デイリータスク.md`（毎日やること）
- `未来メモ.md`（先の日付に紐づけたメモ）
- 前日のデイリーノートの「翌日への引き継ぎ」

これらを自動でかき集めて、今日のタスクリストを作ってくれます。デイリータスクはこんな内容です。

```md
- [ ] おむつ替え
- [ ] 沐浴
- [ ] 料理
- [ ] 掃除
- [ ] 洗濯
- [ ] 猫遊び
- [ ] 15分読書
- [ ] ラジオ体操
```

育休中でも「育児以外にやること」はあります。自分の習慣（読書・ラジオ体操）をデイリーとして登録しておくことで、毎朝自動でリストに入ってきます。

中でも**未来メモ**が育休中にかなり活躍しました。育休中は初めてのイベントが次々と出てきます。健診・役所手続き・予防接種・自分の健康診断・友人の誕生日…。そのたびに「忘れないうちにとりあえずここに書く」というルールにしたら、頭の中がすっきりしました。

育休中の未来メモはこんな感じでした。

```md
## 2026-04-27
- 📦 荷物到着後 → 区役所へ（出生届・マイナンバー手続き）
- 産後の手続き整理（健康保険加入、児童手当申請、乳幼児医療証）

## 2026-05-01
- 🦷 歯医者に電話して予約（仮歯が取れた）

## 2026-05-04
- 🚗 妻の皮膚科送迎（09:00〜）
- 📋 会社提出書類作成・提出

## 2026-05-06
- 👥 友達の面会

## 2026-05-08
- 🏥 2週間検診

## 2026-05-10
- 🎁 母の日プレゼントを買う（妻へ）

## 2026-05-19
- 午前中　友達の面会

## 2026-05-24
- ⛩️ お宮参り

## 2026-05-25
- 🏥 1ヶ月検診
```

出生届・各種行政手続き・妻の通院送迎・友人の面会・お宮参り…育休中の初めての予定がどんどん入ってきます。これを全部「その日になったら勝手に出てくる」状態にしておけるのが未来メモの強みです。その日の朝に `start-daily` を実行すると、該当日のメモが自動でその日のタスクリストに反映されます。自分でリマインダーをセットしなくていいし、「あの予定いつだっけ」とノートを探し回る必要もない。

そして実はここが一番大事なポイントなのですが、**未来メモへの記入も含めて、自分でファイルを直接編集したことは一度もありません。**

「来週の火曜に2週間検診があるんだけど、未来メモに入れておいて」

と Cowork に話しかけるだけ。あとは Claude が `未来メモ.md` の該当日付に書き込んでくれます。テキストエディタを開く必要もなければ、Obsidian でノートを操作する必要もない。**全部チャットで完結**しています。

Google カレンダーを開いてぽちぽち入力するより、チャットで一言伝えるだけの方が断然楽でした。

**② 今日の夕ご飯レシピを自動で取得する**

あらかじめ week-1〜week-5 の5週サイクルでレシピを管理していて、今日の曜日に合ったレシピを自動で引っ張ってきます。

レシピの元ネタは **[memiの1か月食費2万円 節約ワンプレートごはん](https://www.amazon.co.jp/dp/4522438907)** という本です。節約・時短・ワンプレートという育休中にぴったりのコンセプトで、この本を参考に献立を組み立てました。

本の内容はスキャンして Obsidian にドキュメント化し、レシピ・週一の買い物リスト・仕込みリストをそれぞれノートとして整備。それをそのまま `start-daily` スキルの参照先として使っています。

![Obsidianのレシピフォルダ構成](/images/parental-leave-cowork-start-daily/obsidian-recipes-tree.png)

食事周りはそもそも**判断する場面をとことん減らす**方針にしていました。

- **朝ごはん**：味噌汁・目玉焼き・ご飯・納豆で固定
- **昼ごはん**：前日の夜ご飯の残りを弁当にする
- **夜ごはん**：スキルが今日のレシピを自動で出してくれる

朝と昼は考えることがゼロ、夜だけスキルに聞けばいい。「今日何作ろう」という判断コストが完全になくなります。育休中は頭の中が常に赤ちゃんのことでいっぱいなので、食事の選択をこれだけシンプルにできたのは大きかったです。買い物も週一回、リスト通りに買うだけなので、赤ちゃんを連れての外出もシンプルになりました。

**③ ピヨログ（育児記録アプリ）のデータを集計する**

前日の授乳回数・ミルク量・おしっこ・うんち・お風呂の時刻を Google Drive から取得し、さらに過去1週間の週平均も算出してくれます。

ピヨログデータの Google Drive へのアップロードは、こちらの記事を参考にしました。

https://zenn.dev/big_tanukiudon/articles/52d5a4b653eb47

当初は MacroDroid を使って毎晩23:55に自動エクスポートする仕組みを組んでいたのですが、3日目あたりから定期実行がうまく動かなくなってしまいました。原因の深追いよりも育児優先ということで、今はスキル実行前にピヨログアプリから手動でアップロードするようにしています。「完全自動化」とはいかなかったものの、1アクション追加するだけなので実用上は問題なしです。

![ピヨログのGoogle Driveフォルダ](/images/parental-leave-cowork-start-daily/piyolog-google-drive.png)

完母だと授乳は妻に任せるしかないのですが、「昨日の授乳何回だった？ミルクは？」と毎朝聞くのも申し訳なくて。でもサマリーが自動で出るようになってからは、自分で把握できるようになりました。

このデータが使えるようになって、特によかったのが**1日の流れを可視化できること**です。過去の授乳・おむつ替えのタイミングがデータとしてあると、「今日の午前中は10時〜12時あたりが比較的落ち着きそう」という見通しが立ちます。その隙間に買い物を入れたり、レシピの仕込みを済ませたり、ちょっとした作業を入れたりと、**赤ちゃんのリズムに合わせて自分のタスクを組み込む**ことができるようになりました。

実際のサマリーはこんな感じです。

```
| 項目       | 昨日 (5/23)                      | 週平均 (5/17〜5/23) |
| 母乳       | 左27分 / 右37分                  | -                   |
| 授乳回数   | 10回 ↑                          | 8.8回               |
| ミルク     | 0ml                              | -                   |
| 搾母乳     | なし                             | -                   |
| おむつ替え | 8回 ↑（夫7回 / 妻1回）          | 7.2回               |
| お風呂     | 17:45（夫:沐浴 / 妻:保湿ケア）  | -                   |

🌸 noriへ: お宮参りという大切な日に、おむつ替え7回＋沐浴をそつなくこなせましたよ！
大活躍の一日でしたね。うんちが多めだったので、おしりの保湿ケアも引き続きセットでしてあげよう！
```

週平均との比較（↑↓→）やパパへの一言アドバイスも生成されるので、「今日は授乳が多めだったからゆっくり休んでもらおう」という判断がデータとしてできるのがよかったです。

**④ 授乳・おむつのタイムラインを自動生成する**

前日のピヨログデータをもとに、過去平均から授乳・おむつのタイミングを予測してタイムラインに並べてくれます。実際のノートはこんな感じです。

```
| 02:30 | 🍼 授乳（目安）         | 過去5日中3回 ※02時台  |
| 09:40 | 🍼 授乳（目安）         | 過去5日中3回 ※09時台  |
| 12:31 | 🍼 授乳（目安）         | 過去5日中3回 ※12時台  |
| 15:33 | 🍼 授乳（目安）         | 過去5日中3回 ※15時台  |
| 18:06 | 🛁 沐浴（目安）         | 過去3日平均           |
| 18:26 | 🍼 授乳（目安）         | 過去5日中4回 ※18時台  |
| 21:25 | 🍼 授乳（目安）         | 過去5日中4回 ※21時台  |
```

「過去5日中3回 ※02時台」のような根拠付きで時刻が入るので、今日も同じ時間帯に動いてあげやすくなります。これが Google カレンダーにも自動登録されるので、スマホを見るだけで「次の授乳は何時ごろ」が分かります。

![Googleカレンダーに授乳目安・沐浴が自動登録された画面](/images/parental-leave-cowork-start-daily/google-calendar.png)

**⑤ デイリーノートとして保存する**

上記の内容が1つの Markdown ファイルにまとめられて保存されます。

保存先は Obsidian の Vault 内のフォルダに設定しているので、Cowork が作ったノートをそのまま Obsidian でプレビューできます。テーブル形式の育児サマリーやタイムラインもきれいにレンダリングされるので、朝イチで開いたときに「今日の全体像」がひと目で把握できます。

Obsidian のデイリーノート機能と組み合わせることで、過去のノートへのリンクも自然につながり、「先週の子どもの様子どうだったっけ」という振り返りもしやすくなります。

![Obsidianでデイリーノートをプレビューしている画面](/images/parental-leave-cowork-start-daily/obsidian-daily-note.png)

**⑥ Google カレンダーに自動反映する**

タイムラインのイベント（授乳目安・沐浴）とタスクが自動でカレンダーに登録されます。時間指定ありのものは時間指定イベント、タスクは終日イベントとして入ります。

## 実際の使い方

朝、子どもが少し落ち着いたタイミング（大体授乳後）に Cowork を開いて、こう打つだけです。

```
/start-daily
```

あとは Claude が自動で動いて、2〜3分後にはデイリーノートが完成しています。

**作ったノートは、その日1日の行動の軸になります。**

- 朝イチで Obsidian を開き、今日のノートをプレビューして全体像を把握
- タスクが全部消化できているか、隙間時間にチェック
- 料理のときは PC を横に置いて、ノートに展開されているレシピの手順を見ながら調理

特にレシピは材料・作り方まで全部ノートに展開されているので、料理中に「次どうするんだっけ」と別のアプリを開く必要がありません。片手に赤ちゃんを抱えていても、PC 画面さえ見えれば料理できます。

![レシピ通りに作った夜ご飯](/images/parental-leave-cowork-start-daily/dinner.jpg)

ノートには今日やること・レシピ・昨日の育児サマリー・授乳スケジュールがまとまっているので、「今日何するんだっけ」「夜ご飯どうしよう」「昨日のうんちの回数は」といった問いに、全部ここで答えが出ます。

## スキルは自分でカスタマイズできる

`start-daily` は最初から用意されているわけではなく、自分で作ったスキルです。

Cowork のスキル機能は、SKILL.md という Markdown ファイルに「こういう手順でやって」と書くだけで作れます。自分の生活に合わせて少しずつ育てていくのが楽しくて、今もちょこちょこ改良しています。

たとえば最初はレシピ機能がなく、後から「夕ご飯の判断コストを下げたい」と思って追加しました。ピヨログ連携も「データ見るのが面倒」と感じてから追加した機能です。

「不便だな」と思ったら、その日のうちにスキルをアップデートできる柔軟さが、Cowork の好きなところです。

## 育休中だからこそ刺さった

育休中は「やることは多いけど、まとまった集中時間が取れない」というジレンマがあります。

そういうときに、毎朝5分未満で今日の全体像が整理される仕組みがあると、頭の中のノイズがかなり減ります。「あれどうするんだっけ」を考える時間がなくなる、という感じです。

子どもの昼寝の間など、隙間時間で作業するときも「今日のタスクリスト」がすでにある状態から始められるので、立ち上がりが速い。

育休中に限らず、「バタバタしがちな朝を整えたい」という方にはかなりおすすめです。

## まとめ

- Cowork は Claude のデスクトップアプリで、カスタムスキルで繰り返し作業を自動化できる
- `start-daily` スキルで毎朝のデイリーノート作成・カレンダー反映を自動化している
- 育休中の「判断コストを下げたい」というニーズにめちゃくちゃフィットした
- スキルは自分でカスタマイズできるので、生活の変化に合わせて育てていける

育休中のパパ・ママ、あるいは毎朝の段取りを改善したい方はぜひ試してみてください🍼
